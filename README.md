## Repository Structure

Each folder represents a LeetCode problem and usually contains:

- `README.md`: problem description and notes
- `solution.js`: JavaScript solution implementation

Current examples:

- `1twoSum/`
- `21mergeTwoLists/`
- `657robotReturn/`

## Tech Stack

- Language: JavaScript
- Runtime: Node.js

## How To Run Locally

1. Clone the repository.
2. Enter the repository folder.
3. Run a specific solution file with Node.js.

Example:

```bash
node 1twoSum/solution.js
```

## Study Workflow

For each problem, I try to follow this process:

1. Understand constraints and edge cases.
2. Build a brute-force approach first.
3. Optimize the solution.
4. Validate with test examples.
5. Record complexity and notes.

## Complexity Policy

Whenever possible, each solution should include:

- Time complexity analysis
- Space complexity analysis
